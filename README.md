<h3>2️⃣ What is the spread operator (...)</h3>
<p>Ans: The spread operator(...) in JavaScript is a syntax that allows an array or string to be expanded into individual elements, or an object to be expanded into key-value pairs. It is very useful for copying, combining, or passing elements without modifying the original structure.</p>

<h3>3️⃣ What is the difference between map(), filter(), and forEach()?</h3>
<p>Ans: map(), filter() and forEach() in javascript are array methods but with a difference. map() function is used to map each element of an array and return a new array with the updated values without modifying the original array. filter() Returns a New Array, and Filters the Elements. For example, while the method map() is used to transform an array into a new one according detailed mapping function that you provide for each element in the array, and then returns this new array as result of operation; on other hand, method forEach() executes callback function for each element. In other words, map() and filter() results in a new array (also called the return array) and forEach() is used to perform actions without creating a return array.</p>

<h3>4️⃣ What is an arrow function?</h3>
<p>Ans: An arrow function in JavaScript is a shorter, simpler way to write a function with the => syntax. It was introduced in ES6 to make functions easier to write and read. Arrow functions are particularly helpful for simple operations or callbacks.</p>

<h3>5️⃣ What are template literals?</h3>
<p>Ans: Template literals in JavaScript are a modern way to create strings using backticks (`) instead of single or double quotes. They make it easy to embed variables or expressions directly into a string using the ${...} syntax, known as interpolation. This lets you create dynamic strings without needing to use complicated string concatenation. Template literals also support multi-line strings, so you can write text across several lines without using special newline characters. Overall, they make string creation in JavaScript more readable, flexible, and convenient, especially when combining variables, expressions, or longer text blocks.</p>
